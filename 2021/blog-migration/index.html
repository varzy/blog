关于这个博客的一些变更

Aiden's Blog
2021-03-15
Essay
blog, original, recommend

一句话总结：这里不会再更新技术文章了，而是只保留我的思索、臆语和长文字。以前的和未来的技术文章都会统一发布在 Notion 上的笔记页面。

大学的时候我开始搞博客，最开始的目的就是分享一些自己的代码笔记，以及一些因为一时兴起而瞎写的长文字。但奈何我总是想得很多，做得很少。玩博客至今五六年，换了四五个博客平台和方案，但输出的文章加起来不过百篇，还大部分都被我移到了 “abandoned” 和 “backup” 文件夹。

中间有很多次都下定决心好好更博，然后三分钟热度地更新了过期的依赖，重新找了满意的主题，结果第二天就把更新这件累人的事情抛之脑后了。我当然知道对于博客来讲最重要的是内容，但我总是把工作停留在并不值得关心的 “仪式感” 上。

那么这一次，我又做了什么呢？

  1. 我移除了博客上所有的技术文章
  2. 我在 Notion 上公开了自己的技术笔记，未来所有的技术文章都会直接在 Notion 更新

在此之前，我产出内容的平台多且杂。比如心情、总结之类的严肃长文字会发布在博客上，小的代码片段、笔记都会放在 Notion 上，而其中比较值得拿来一讲的内容，也会加以整理在在这个博客上发布。但这样会有几个问题。

首先就是发布博客的成本是非常高的。由于我（目前）还在使用 Hexo 作为博客平台，因此每次修改内容后都需要重新编译成静态文件并发布到 Github。纵使现在有 hexo-deployer-git 插件和 Github Actions 加持，但改一个错别字就要重新走一遍发布流程实在是过于繁琐。

其次，我无法保证笔记和博客的同步。如果某篇笔记有更新，我可能并不会记得同步更新自己的博客，这就有可能导致博客上的内容早已过时。而把 Notion 笔记公开后，我只需要像往常一样编辑笔记就行了，非常方便。

至于我为什么到现在才把笔记公开呢？说来惭愧，我的笔记大部分都是随手一记，内容的质量并不高。从博客之前技术文章的数量就可见一斑。这次我也是对笔记进行了一次大整理，虽然里面的内容还有很多 “偏难怪”，但也不乏一些我的愚思和拙见。即使内容不咋滴，但希望帮助他人的心情是实实在在的。

那么如何访问我的笔记呢？

  1. 你可以直接访问 Notion 页面：https://www.notion.so/Notes-23a151f7310543e186304f74c8b15034。但这种方式并不被推荐，毕竟我也无法保证这么长的 url 会不会什么时候就失效了
  2. 访问本域名下的笔记链接：https://varzy.me/notes
  3. 先访问我的个人主页，里面有一切可访问的链接和我的联系方式：https://varzy.me

说句题外话。关于方式 2，本来我是想添加一个子域名做一个 302 跳转，但阿里云的域名如果不备案就不能进行跳转。。。所以我只好在 Github Pages 上建立了一个空页面，用 js 做了一个跳转。

3 月 14 号，也就是发表这篇文章的前一天，我又给自己立了个 flag：保证自己每一周都至少产出一篇博客或技术笔记。这确实是个几乎不可能完成的 flag，但我希望尽可能坚持久一点吧，不要再是三分钟热度了（笑）。
