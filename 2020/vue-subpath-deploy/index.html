Vue 项目在子路径部署的两种方案

Aiden's Blog
2020-01-14
Code
deploy, frontend, original, tutorial, vue, vue-cli, vue-router

方案一：hash 模式

xxx.com/app1/#/index

  • 路由路径直接被前端接管，# 后面的是前端路由
  • 优点是部署简单
  • 缺点是 url 比较反直觉，也不美观

vue.config.js

router.js

nginx.conf

方案二：history 模式

xxx.com/app1/index

  • 路由都走到 nginx，再由 nginx 转发回前端的入口文件
  • 优点是 url 好看，符合直觉
  • 缺点是需要配合 Web Server 才能实现

router.js

nginx.conf
